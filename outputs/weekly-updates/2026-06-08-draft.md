- **Jun 9 design session** — face-to-face compliance workflow, AMR family visit plans, and client profile generation confirmed with PCFS ongoing services team:
  - **Face-to-face (F2F) visit logging:** workers log visit date, who conducted it, location, one-on-one status, and upload the signed F2F form (caregiver, child, worker, and supervisor signatures required); list is filterable by visit type
  - **F2F compliance deadlines:** children in care must be seen by the 20th of each month, with a hard deadline on the business day before the last Monday of the month; system triggers high-priority alerts for overdue visits
  - **Override permissions:** supervisors, CRDs, and admin (including Kelvin) can override default visit frequency pathway; override is logged with a mandatory reason; customer care agreements may require less frequent visits and system must accommodate CRD pathway adjustments
  - **Visit conductor selection:** system captures the actual person who conducted the visit (not just role); case aides may conduct visits but lack system access — the responsible worker uploads documentation
  - **Case note deadline:** workers have five days after a visit to submit case notes; system triggers reminders if notes are missing; freeform text entry, document upload optional
  - **Collateral F2F workflow:** when a visit is overdue, worker can request a collateral visit — triggers a 24-hour supervisor approval task; CRDs can reassign if supervisor is unavailable; collateral form requires worker and supervisor signatures; if signature cannot be obtained, a checkbox + reason field triggers CRD approval for the exception
  - **AMR family visit plans:** visitation schedules created with frequency, duration, location, and authorized participants; agency-generated schedules uploaded as documents (system-generated schedules may not cover all scenarios); plan versioning tracks history, reasons for updates, and associated signed documents; individual visitations logged with schedule, date, duration, location, supervising staff, and present family members
  - **No Contact Orders (NCOs):** any NCO involving anyone in the child's genogram or family group displayed prominently on the case file and child's profile
  - **Client profile document generation:** Leanne to provide youth profile and annual assessment templates; system generates PDF for case transfers and new placements; document is printed only — not emailed or shared electronically; each generation is logged (user + timestamp) for privacy compliance
  - **Task automation:** system-triggered tasks (overdue visits, missing notes) can auto-assign or allow worker to select due date and assignee; temporary worker assignment flagged for a future release
- Wireframes and triggered-task Excel document to be distributed by Gabrielle for team review; Leanne to provide dropdown field values for AMR visit frequency and location
