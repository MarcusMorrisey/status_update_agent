## I5 B – Abuse Investigations

**Features**

- Triggers and integration with Intake module
- Investigation workflow
- Investigation document management

**Weekly Progress**

- Two design sessions held Apr 28 with Kelvin (PCFS abuse team lead), Sherry (abuse supervisor), Marcus, and Ross — full investigation workflow confirmed:
  - **Intake trigger:** abuse categories (sexual abuse, physical abuse, neglect, emotional abuse) auto-notify abuse team; worker also gets a pop-up prompt confirming the report will be sent
  - **Triage screen:** abuse team sees incoming referral with disclosure content, originating intake/case reference, and access constraints; options are Open Investigation, Reject (with mandatory reason), or Return for More Information (pending status — 24-business-hour clock restarted when info received)
  - **Form routing:** Peguis form for Peguis member offenders; ANCHOR form for Winnipeg/Headingley/East & West St. Paul non-member offenders; all others on Peguis form — routing confirmed as Sherry/Kelvin decision point
  - **Investigation lifecycle phases:** Intake Received → Assessing → Active Investigation → Finding/Closure; PI Unit branch added for provincial investigation referrals
  - **Timers:** 24-business-hour windows at two stages (initial report to abuse team; referral form back to worker); override capability for committee deadlines; business-hour/holiday calendar required
  - **Confidentiality tiers confirmed:** Standard (investigators + supervisor + director); Confidential (abuse team only, reasons: high-profile criminal matter, staff involvement, public official, media, civil legal); Restricted (Sherry + Kelvin only — used for child death cases)
  - **Evidence management:** no sensitive content (DVDs, CSAM) stored in Waabanong; reference-only logging with case number and external system (e.g., evidence.com, RCMP file number)
  - **Audit timeline:** append-only log; modifications create a superseding entry rather than deleting prior content
  - **Access management:** explicit grant-per-person model; Sherry/Kelvin manage access; investigators can freeze/unfreeze reporting on police request
  - **Child abuse committee:** monthly; only Peguis-member offenders presented; outcome logged as single entry (date presented, committee decision)
  - **Auto-generate investigation number:** Waabanong to auto-assign PCFS-formatted sequence number (e.g., 2026-015) on "Accept and Open" — removes manual clerk step
  - **Finance Assigner role discussed:** similar queue-based model confirmed for abuse coordinator role (Sherry/Kelvin see pending intakes queue, assign investigators, monitor timer breaches)
- Wireframes walked through in Part 2 session; feedback captured on rejection flow wording, phase transition permissions, evidence reference categories, and information disclosure logging

## I7 - Financial Tracking & Exports for Sage

**Features (Potential scope may include)**

- Intake-level emergency supports
- Case-level budgets
- Spending caps and approvals
- Export files to import into Sage 300

**Weekly Progress**

- Finance design session held Apr 28 with PCFS finance team (Barry, Bev, Christina) and Ross; demo environment crashed at session start — session continued on whiteboard/verbal walkthrough:
  - **PO workflow confirmed:** worker submits purchase request (vendor, case, amount) → supervisor approves/rejects (with reason) → finance clerk reviews → senior finance authorizes → Waabanong generates PO document → worker makes purchase and attaches receipt → finance reconciles and exports to Sage
  - **Credit card reconciliation approach:** Barry exports RBC statement (CSV); Waabanong imports and auto-assigns transactions to card holders by last-four-digit matching; each card holder receives task to attach receipt and allocate to case/budget; supervisor reviews; finance reconciles and exports to Sage — removes current email-and-spreadsheet process
  - **Finance Assigner role confirmed:** a small group (Bev/Barry) will assign finance tasks from a shared queue rather than direct assignment; load-balancing view (task counts per clerk) to be included in dashboard
  - **Expense categorization:** prevention / maintenance / operations — all reconciliation tracked in Waabanong regardless of category; non-child-services charges flagged separately in export report
  - **Global holiday calendar requirement confirmed** — needed for accurate business-hour timer logic across finance and abuse workflows (A-152); Ross to implement (also flagged in abuse session)
